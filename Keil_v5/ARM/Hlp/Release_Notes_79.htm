The Microcontroller Development Kit (MDK-ARM) supports software development for and debugging of ARM7 and ARM9 processor-based devices. A detailed description of MDK-ARM can be found in the section Complete User's Guide Selection of the µVision window Books. The Getting Started is a good introduction to the main concepts of MDK-ARM and gives guidelines for programming ARM devices.

  • [Middleware]

MDK Version 5.13 Legacy support for ARM7 & ARM9 devices

  • [Middleware]
    • [ARM Middleware]
      • Added: RTX_CR4_B.lib, RTX_CR4_BA.lib, RTX_CR4F_B.lib, RTX_CR4F_BA.lib, RTX_CR4F_L.lib, RTX_CR4F_LA.lib, RTX_CR4_L.lib, and RTX_CR4_LA.lib RTX libraries for Cortex-R4 and Cortex-R4F in ARM as well as in thumb mode. For naming consistency the current RTX_CR4.lib is now replaced by RTX_CR4_L.lib.
        The following naming convention is used for the above libraries: L = little endian, B = big endian, A = ARM mode (else thumb), and F = HW Floating Point enabled.

MDK Version 5.12a Legacy support for ARM7 & ARM9 devices

  • [Middleware]
    • [ARM Middleware]
      • Added: Middleware libraries build with ARM7/ARM9 ARM mode enabled, indicated by the letter 'A' in the library name (e.g. FS_ARM_LA.lib).
      • [FlashFS]
        • Corrected: when a embedded drive is full, the system fails to write allocation information and correctly close a file. So the drive becomes corrupted.
        • Corrected: EFS fails to detect full blocks during fopen in case of a RAM drive.
    • [Segger Graphic Library]
      • Updated: Segger emWin graphic library version 5.26a in folder ..\ARM\Segger\emWin\.

MDK Version 5.12 Legacy support for ARM7 & ARM9 devices

  • [Middleware]
    • [ARM Middleware]
      • [FlashFS]
        • Added : basic support for eMMC devices (MMC NAND flash).
        • Corrected: multithread file write function which could cause corruption of the EFS file system.
        • Corrected: an frename function issue when using FAT name cache. After renaming a file it was still possible to open it using old file name.
        • Corrected: timestamp handling when the file size remains unchanged after a write access.

        • Removed: USB Host OHCI delay of 1 ms when returning from an interrupt service routine.
      • [RTX]

Device Database™

A unique feature of the Keil µVision5 IDE is the Device Database™ which contains information about more than 3000 supported microcontrollers. When you create a new µVision5 project and select the target chip from the database, µVision5 sets all assembler, compiler, linker, and debugger options for you. The only option you must configure is the memory map.

As new devices become available, they are added to the database along with data sheets and header files.

Peripheral Simulation

The µVision5 Debugger provides complete simulation for the CPU and on-chip peripherals of the most popular ARM-powered microcontroller devices. To discover which peripherals of a device are supported, in µVision5 select the Simulated Peripherals item from the Help menu. You may also use the web-based Device Database™. We are constantly adding new devices and simulation support for on-chip peripherals so be sure to check Device Database™ often.
