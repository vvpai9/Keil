Getting Started with the GNU Compilation Tools

Development Studio 5 includes a distribution of the GNU Compiler Collection (GCC) and related GNU utilities. This distribution is referred to in this document as the 'GNU Compilation Tools'.

The GNU Compilation Tools can be used to build applications and libraries suitable for ARM Linux targets, including the example ARM Linux distribution which can be found in the DS-5 examples.

The GNU Compilation Tools are a subset of the 13.03 release of the Linaro distribution for ARM GNU/Linux and are based on GCC version 4.7. Sources can be found in the 'sources' directory of your installation medium, on the DS-5 download page and at Linaro's web site.

Terminology

This document makes use of the following terms.

TermMeaning
HostThe computer used for building and debugging.
TargetThe system on which the built software is deployed and run.
Target distributionThe Linux distribution running on the target system.
SysrootA directory on the host system containing a copy of the files that exist in the target distribution and/or extra files required to develop for the target distribution.

Host Development Tools

GNU Compilation Tools are in the bin directory. They can be identified by the prefix arm-linux-gnueabihf-. The tools can be used from the command line, or in Makefile projects in the Eclipse IDE. For Windows users, the DS-5 Command Prompt has the tools in its path.

Some of the most important tools are:

arm-linux-gnueabihf-arGNU Librarian
arm-linux-gnueabihf-asGNU Assembler
arm-linux-gnueabihf-gccGNU C Compiler

gdbserver

A gdbserver utility is included as part of the GNU Compilation Tools, under the arm directory of your Development Studio 5 installation. It is taken from Linaro GDB 2013.03 and corresponds to GDB 7.5. It is suitable for use on ARM Linux targets with a VFP and Advanced SIMD coprocessor.

Libraries And Headers

Sysroots

Applications and shared libraries for target distributions must be built with respect to resources available on the target distribution. In particular, libraries and corresponding headers from the target distribution must be available on the host. The necessary libraries and headers are here referred to as a sysroot. A sysroot may or may not also include binaries, configuration files, or any other part of the target distribution.

Sysroot Selection

You should build your code with respect to the headers and libraries for your target distribution. However, three sysroots are provided with DS-5:

  1. Hardfloat Linaro sysroot in the installation beneath: sw/gcc/arm-linux-gnueabihf/libc
  2. Softfloat Linaro sysroot in the installation, also beneath: sw/gcc/arm-linux-gnueabihf/libc
  3. Softfloat CodeSourcery sysroot on the installation medium and the DS-5 download page at: linux_distributions/Linux_distribution_example.zip

The hardfloat Linaro sysroot is suitable for building applications to run on hardfloat targets such as Linaro LEBs and is used by default.

The softfloat Linaro sysroot is suitable for building applications to run on softfloat targets and is used when -marm -march=armv4t -mfloat-abi=soft are specified and --sysroot is left unspecified.

The softfloat CodeSourcery sysroot is suitable for building applications to run on softfloat targets such as the Linux distribution provided for the DS-5 examples. It is in the zip file at distribution/filesystem/armv5t_mtx and may be used by specifying the flags: -marm -march=armv4t -mfloat-abi=soft --sysroot=/path/to/unzip/dir/distribution/filesystem/armv5t_mtx. The softfloat sysroot is taken from the 2009q3-67 GNU/Linux release of CodeSourcery G++ Lite, available from Mentor Graphics Sourcery Tools. Note that this sysroot's libraries are not guaranteed to be debug-compatible with the example distribution.

To use your own sysroot, specify its location on your host using --sysroot. If it is a softfloat sysroot, also set the flags -marm -march=armv4t -mfloat-abi=soft.

Note

Ordinarily, you are able to install sysroot filesystems onto real targets. However, if you are using a Windows host then the provided sysroots will not be suitable for use on a real target, due to differences between Windows and Linux filesystems.
